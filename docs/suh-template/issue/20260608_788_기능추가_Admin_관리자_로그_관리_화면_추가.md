📝 현재 문제점
---

- 운영 중 서버 로그를 확인하려면 매번 SSH로 서버에 접속해 `/mnt/romrom/logs/` 경로로 들어가서 `tail`, `grep`, `zcat`, `scp` 등을 직접 수행해야 한다.
- 에러 로그를 모아 보거나, 특정 키워드를 검색하거나, 과거 롤링된 `.gz` 파일 내용을 확인하는 작업이 모두 수작업이라 번거롭고 느리다.
- 기존에 만들어 둔 SSE 디버그 로그 스트림(앱 테스트 빌드 플로팅 버튼용, `/api/app/debug/log-stream`)이 있으나 용도가 다르고 현재 정상 동작하지 않는다.

🛠️ 해결 방안 / 제안 기능
---

- 관리자 웹(`/admin/logs`)에 **로그 관리 섹션**을 추가하여, 서버에 직접 접속하지 않고도 로그 관련 작업을 화면에서 모두 처리한다.
- 로그는 이미 파일(`romrom.log` + 날짜별 `.gz` 롤링)로 쌓이고 있으므로, 별도 DB 적재 없이 **파일을 읽어서** 제공한다.
- 화면 구성 (탭 4개):
  - **조회·검색**: 현재 로그 최신순 조회 + 레벨 필터(ERROR/WARN/INFO/DEBUG) + 키워드 검색 + 줄 수 선택
  - **에러 대시보드**: 최근 N분 ERROR/WARN을 예외 클래스별로 집계(발생 횟수 / 마지막 발생 시각 / 대표 메시지)
  - **실시간**: `tail -f` 처럼 라이브 출력 (기존 SSE 인프라 재활용)
  - **파일/다운로드**: 로그 파일 목록 추적(현재 `.log` + 과거 `.gz`, 크기/수정시각) + 로그 총 용량·디스크 상태 + 시간범위(최근 5분/1시간/6시간/24시간) 다운로드 + 파일 통째 다운로드 + `.gz` 압축 해제 후 화면 조회/검색
- 기존 SSE 디버그 스트림을 이번 관리자 화면으로 **통합**한다. (앱 전용 `/api/app/debug/log-stream` → 관리자 인증 경로 `/api/admin/logs/stream`로 일원화, 기존 Broadcaster/Appender 재활용)
- **브라우저 부하 최소화**가 필수 요구사항이다.
  - 실시간 탭: DOM 노드 상한 500줄(ring buffer), 탭 활성 시에만 SSE 연결(`visibilitychange`), `requestAnimationFrame` 배칭
  - 조회/검색: 줄 수 캡, 무한 로드 금지
  - `.gz`: 서버에서 압축 해제 후 필요한 라인만 전송 (브라우저로 통째 전송 금지)

⚙️ 작업 내용
---

- `LogFileService` 신규 (RomRom-Application): 파일 역방향 tail / 에러 집계 / 파일 목록·디스크 상태 / 시간범위 추출 / 파일 다운로드 / `.gz` 조회
- `LogLineParser` 신규: 로그 라인 파싱(logback 포맷 1:1), 멀티라인 스택트레이스 결합
- `AdminApiController` 엔드포인트 추가 (query/errors/files/gz-query/download/download-file/stream) + 기존 admin 컨벤션(POST + multipart, `AdminRequest`/`AdminResponse`) 준수
- `AdminRequest`/`AdminResponse` 로그 관련 필드 추가
- `SecurityUrls.ADMIN_PATHS` 경로 등록 + **path traversal 방어**(화이트리스트 검증)
- `AdminPageController` + `layout.html` 사이드바 메뉴 + `admin/logs.html` (Thymeleaf + DaisyUI + 바닐라 JS)
- `*ControllerDocs` Swagger 문서화 + `@ApiChangeLog` 추가
- 기존 `DebugController` SSE 통합 정리

> 범위 제외(YAGNI): DB 로그 적재, `.gz` 시간범위 다운로드, 로그 자동 마스킹, 로그 파일 삭제, 다중 서버 통합
>
