📝 현재 문제점
---

- `POST /api/admin/members/detail` 응답에는 회원 기본정보 + 소유 물품 + 신고당한 기록 정도만 포함되어 있다.
- 해당 회원의 **거래내역(TradeRequestHistory), 채팅방 목록, 본인이 신고한 이력, 제재이력**은 별도 API를 호출해야 하거나 아예 노출되지 않는다.
- 운영자가 특정 회원의 문제 상황을 추적하려면 화면을 여러 번 이동해야 하며, "이 회원이 어떤 회원들과 거래/채팅했는지" 같은 cross-member 관계 추적이 불가능하다.
- 회원 상세 화면에서 정지/해제 외에 강제탈퇴, 보유 물품 일괄 삭제 등 운영성 액션도 한 화면에서 처리할 수 없다.

🛠️ 해결 방안 / 제안 기능
---

회원 상세 화면을 "회원 360 뷰"로 재설계한다. 한 응답(또는 명확히 분리된 페이지네이션 sub-endpoint) 안에서 다음을 모두 확인할 수 있어야 한다.

- 회원 기본정보 (현행 유지)
- 소유 물품 목록 (현행 유지)
- **거래내역**: 본인이 요청한 거래 + 본인 물품에 들어온 거래, 각 거래의 상대방 회원 deep-link
- **채팅방 목록**: 회원이 참여 중인 채팅방, 상대방 회원 deep-link
- **신고 이력**: 신고당한 기록 + 본인이 신고한 기록
- **제재이력**: 정지/해제 히스토리

회원-회원 연결은 거래/채팅 항목에서 상대방 memberId를 노출해 admin UI가 deep-link로 이동할 수 있게 한다.

추가 액션 API:
- 강제 탈퇴 (현재 status 변경 + 회원탈퇴 cascade 로직 재사용)
- 보유 물품 일괄 삭제 (선택 itemId 배열)
