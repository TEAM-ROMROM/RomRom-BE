📝 현재 문제점
---

- 거래(Trade/TradeRequestHistory) 도메인은 현재 admin 대시보드에서 **진행중 거래 카운트**만 노출되고 있다.
- 거래 목록 조회, 거래 상세(양쪽 회원/물품/채팅방/진행상태), 거래 강제 취소/완료 처리를 위한 admin API가 전무하다.
- 거래 분쟁 발생 시 운영자가 개입할 수단이 없어, 채팅 신고 → 회원 정지로 우회 처리하는 비효율이 발생한다.

🛠️ 해결 방안 / 제안 기능
---

거래 도메인에 admin API 신규 추가.

- 거래 목록 조회: 상태(PENDING/ACCEPTED/COMPLETED/CANCELED 등) 필터, 기간 필터, 회원/물품 검색
- 거래 상세 조회: 요청자/피요청자 회원 요약, 양쪽 물품 요약, 연결된 채팅방, 거래 상태 변경 이력
- 거래 강제 취소 / 강제 완료: 운영자가 사유와 함께 거래를 강제 종료
