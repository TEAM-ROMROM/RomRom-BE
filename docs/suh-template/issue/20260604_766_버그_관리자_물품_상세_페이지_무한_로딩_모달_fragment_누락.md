📝 현재 문제점
---

관리자 페이지에서 물품 상세(`/admin/items/{itemId}`)에 진입하면 **로딩 스피너만 무한 표시되고 상세 내용이 렌더되지 않는다.**

브라우저 콘솔 에러:

```
Uncaught TypeError: Cannot read properties of null (reading 'options')
    at populateDeleteReasons (...)
    at HTMLDocument.<anonymous> (...DOMContentLoaded...)
```

서버는 정상이다. 백엔드 로그상 `AdminPageController.itemDetail`이 `200 OK`로 `admin/item-detail` 템플릿을 정상 렌더하고, `/api/admin/items/detail` API도 응답한다. 100% 프론트엔드(Thymeleaf 템플릿) 버그다.

**근본 원인**

`item-detail.html`에서 `<div layout:fragment="content">`를 닫는 `</div>`가 모달 6개(`editModal`, `hideModal`, `unhideModal`, `deleteModal`, `tradeForceModal`, `suspendModal`) **앞**에서 일찍 닫힌다.

Thymeleaf Layout Dialect는 `layout:fragment="content"` 영역만 레이아웃에 삽입하므로, fragment 밖에 있는 `<dialog>` 모달들은 **렌더 결과에서 통째로 버려진다.** 정상 동작하는 `items.html`은 모달이 content fragment 안에 위치한다.

**증상 사슬**

1. 모달이 fragment 밖 → 렌더 HTML에 모달 미포함
2. `deleteReasonSelect` 등 모달 내부 엘리먼트가 DOM에 없음 → `getElementById` 가 `null`
3. `populateDeleteReasons()`는 null 가드가 없어 `select.options` 접근 시 `TypeError`
4. `DOMContentLoaded` 핸들러가 이 지점에서 중단되어 `loadItemDetail()`이 호출되지 않음
5. 로딩 스피너가 영원히 표시됨

`populateConditionSelect()` / `populateCategorySelect()`는 `!select` 가드가 있어 조용히 통과하지만, `populateDeleteReasons()`만 가드가 없어 폭발한 것이 차이다.

🛠️ 해결 방안 / 제안 기능
---

- content fragment를 닫는 `</div>`를 모달 6개 뒤(마지막 `</dialog>` 직후, `js` fragment 전)로 이동하여 모달이 fragment 안에 포함되도록 한다 (`items.html`과 동일한 구조).
- `populateDeleteReasons()`에 `!select` null 가드를 추가하여, 향후 fragment 구조가 깨지더라도 스크립트가 중단되지 않고 조용히 통과하도록 방어선을 둔다 (`populateConditionSelect`와 일관성).
