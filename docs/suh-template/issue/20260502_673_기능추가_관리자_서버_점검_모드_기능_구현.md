📝 현재 문제점
---

- 서버 배포 또는 장애 발생 시 모든 API 요청에 의도하지 않은 에러가 반환되며, 프론트엔드에서 일관된 점검 화면을 보여줄 수 없음
- 어드민이 서버 점검 상태를 수동으로 제어할 수 있는 기능이 없음
- 점검 중 사용자에게 점검 메시지 및 예상 종료 시간을 안내할 방법이 없음

🛠️ 해결 방안 / 제안 기능
---

- 어드민 화면에서 수동으로 점검 모드를 ON/OFF 할 수 있는 기능 추가
- 점검 모드 활성화 시 어드민 API, 버전 체크 API, 헬스체크를 제외한 모든 API에 503 응답 반환
- 앱 시작 시 호출하는 `/api/app/version/check` 응답에 점검 여부, 메시지, 예상 종료 시간을 포함하여 프론트가 즉시 판단 가능하도록 처리
- 사용 중 점검 모드가 활성화될 경우 503 + `MAINTENANCE_MODE` ErrorCode를 수신한 프론트가 점검 화면으로 전환

⚙️ 작업 내용
---

- `ErrorCode`에 `MAINTENANCE_MODE` (503) 추가
- `SystemConfig` Redis에 점검 관련 설정키 3개 추가
  - `server.maintenance.enabled`
  - `server.maintenance.message`
  - `server.maintenance.end-time`
- `MaintenanceFilter` 추가 (Security Filter Chain 최상단에 배치)
  - 화이트리스트: `/api/admin/**`, `/api/app/version/check`, `/actuator/**`
  - 점검 모드 ON 시 나머지 요청에 503 + `MAINTENANCE_MODE` 즉시 반환
- `SystemResponse`에 점검 관련 필드 3개 추가
  - `maintenanceEnabled`, `maintenanceMessage`, `maintenanceEndTime`
- `SystemConfigService`에 점검 설정 get/update 메서드 추가
- `AdminApiController`에 점검 모드 관련 엔드포인트 2개 추가
  - `POST /api/admin/config/maintenance/get`
  - `POST /api/admin/config/maintenance/update`
- `AdminControllerDocs` Swagger 문서 업데이트
- `AdminRequest` / `AdminResponse`에 점검 관련 필드 추가
