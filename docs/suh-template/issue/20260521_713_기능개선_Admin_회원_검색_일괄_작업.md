📝 현재 문제점
---

- 현재 회원 목록 조회는 계정상태/정지유형 필터 정도만 지원하고, 텍스트 검색은 약하다 (닉네임/이메일/ID/전화 통합 검색 없음).
- 다중 회원을 일괄로 정지/탈퇴할 수단이 없어, 어뷰징 계정군이 발생했을 때 한 명씩 처리해야 한다.

🛠️ 해결 방안 / 제안 기능
---

- 통합 검색: 닉네임/이메일/memberId/전화번호 중 어떤 값으로든 검색 가능
- 다중 선택 일괄 작업: memberId 배열 입력 → 정지/탈퇴 일괄 처리
- 일괄 작업은 트랜잭션 단위로 부분 실패 처리 (개별 결과 응답)

⚙️ 작업 내용
---

- [ ] `AdminMemberService.searchMembers` 강화: 검색어를 닉네임/이메일/ID/전화에 OR 매칭
- [ ] `AdminRequest.searchKeyword` 활용 (기존 필드 재사용)
- [ ] API:
  - `POST /api/admin/members/bulk-suspend` — memberIds + 사유 + 기간
  - `POST /api/admin/members/bulk-withdraw` — memberIds + 사유
- [ ] 응답에 개별 회원별 처리 결과(성공/실패 사유) 배열 포함
- [ ] `AdminMemberControllerDocs` 갱신
- [ ] 통합 테스트 (부분 실패 케이스 포함)
