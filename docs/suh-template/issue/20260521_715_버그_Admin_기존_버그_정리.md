📝 현재 문제점
---

Epic #707 작업 중 발견된 admin 관련 기존 버그/이슈를 정리하여 한 번에 처리한다.

1. **물품 admin 삭제 시 FK 제약 위반 가능성**
   - `ItemService.deleteItemByAdmin` 호출 시 `trade_request_history` → `item` FK 제약 위반 발생 사례
   - cascade 삭제 순서: `chat_room` → `trade_request_history` → `item`

2. **AlertConfig API Content-Type 불일치**
   - `POST /api/admin/alert-config/get` 호출 시 일부 클라이언트에서 500 에러
   - 컨트롤러의 `consumes` 설정 및 multipart/form-data 처리 검토 필요

3. **회원 신고 처리 후 상태 변경 불완전**
   - `members/suspend` 엔드포인트에서 `reportId`가 있으면 신고를 COMPLETED로 변경하나, `reportType` 필수 — 호출자가 빠뜨릴 여지 있음
   - 신고 → 정지 연관관계 자동 추론 또는 명확한 validation 메시지 필요

🛠️ 해결 방안 / 제안 기능
---

- 1번: ItemService에서 admin 삭제 시 cascade 순서 보장. 삭제 전 관련 chat_room / trade_request_history 정리. 통합 테스트로 회귀 방지
- 2번: `consumes`를 `multipart/form-data`로 통일하거나 클라이언트가 보내는 Content-Type 명세 확인 후 처리. Docs 갱신
- 3번: `reportId`만 받고 `reportType`을 서버에서 조회로 추론하거나, 누락 시 명확한 4xx 응답
