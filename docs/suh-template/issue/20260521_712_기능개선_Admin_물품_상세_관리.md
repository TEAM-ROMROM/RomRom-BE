📝 현재 문제점
---

- 현재 admin 물품 API는 목록 조회와 삭제만 가능하다.
- 물품 상세 (이미지, 등록자, 거래 이력, 신고 이력)를 한 번에 볼 수 없어, 신고 처리 시 별도 화면 이동이 필요하다.
- 운영상 부적절한 물품을 "삭제" 외에는 처리할 수 없다 — 잘못된 카테고리/가격 수정, 노출만 차단(soft hide) 같은 중간 단계가 없다.

🛠️ 해결 방안 / 제안 기능
---

- 물품 상세 조회 API 추가: 물품 정보 + 이미지 목록 + 등록자 요약 + 거래 이력 + 신고 이력
- 카테고리/가격 수정 API (운영성 보정용)
- 노출 차단(soft hide) API: 실데이터 삭제 없이 일반 사용자 조회/검색에서 제외

⚙️ 작업 내용
---

- [ ] `AdminItemService`에 다음 메서드 추가:
  - `getItemDetail(itemId)` — 거래/신고 이력 포함
  - `updateItem(itemId, category, price)` — 운영 보정
  - `hideItem(itemId, reason)` / `unhideItem(itemId)` — 노출 차단
- [ ] `Item` 엔티티에 `isHidden` 컬럼 추가 (Flyway 마이그레이션, 멱등성 규칙 준수)
- [ ] 일반 사용자 물품 조회 쿼리에 `isHidden = false` 필터 추가 (ItemRepository, 검색 쿼리, 카테고리별 조회 등 전부)
- [ ] API:
  - `POST /api/admin/items/detail`
  - `POST /api/admin/items/update`
  - `POST /api/admin/items/hide`
  - `POST /api/admin/items/unhide`
- [ ] `AdminItemControllerDocs` 갱신
- [ ] 통합 테스트 (특히 hidden 물품이 일반 사용자 API에서 안 보이는지 검증)
