⚙️[기능추가][채팅] 채팅 읽음 표시 기능 설계 및 구현

📝 현재 문제점
---

- 채팅 메시지를 상대방이 읽었는지 확인할 수 없음
- 카카오톡처럼 읽음 여부를 시각적으로 표시하는 기능 부재
- 백엔드 간 실시간 읽음 상태 동기화 구조 미비

🛠️ 해결 방안 / 제안 기능
---

- 메시지 옆에 "읽음" 텍스트 표시 (카카오톡 숫자 방식이 아닌 텍스트 방식)
- 상대방이 해당 메시지를 포함한 채팅방을 읽었을 때 읽음 상태로 전환
- 읽음 상태는 WebSocket 또는 서버 이벤트를 통해 실시간으로 상대방에게 전달

⚙️ 작업 내용
---

- 채팅 읽음 상태 데이터 설계 (메시지 단위 or 채팅방 단위 마지막 읽음 위치)
- 읽음 이벤트 발생 시 백엔드 처리 로직 명세
- 실시간 읽음 상태 전달 방식 명세 (WebSocket 이벤트 구조 포함)
- 읽음 상태 조회 API 설계
- 채팅방 진입 시 자동 읽음 처리 명세
