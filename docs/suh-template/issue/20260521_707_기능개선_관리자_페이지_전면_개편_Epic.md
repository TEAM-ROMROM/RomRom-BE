📝 현재 문제점
---

- 현재 관리자 페이지는 도메인별로 **단순 조회 수준**의 API만 제공하고 있어, 관리자가 사용자/물품/신고/거래 등 운영 업무를 한 화면에서 통합적으로 처리하기 어렵다.
- 회원 관리에서 해당 회원의 **거래내역, 채팅 이력, 다른 회원과의 연결 관계**를 추적할 수 없어, 운영 이슈 발생 시 도메인을 넘나들며 추적해야 한다.
- 거래(Trade) 도메인은 대시보드 카운트만 노출되고, 채팅(ChatRoom) 도메인은 admin API가 **0개**로 운영 사각지대가 존재한다.
- 신고 처리도 상태 변경만 가능하고, 피신고자 정지/물품 삭제 등 후속 조치를 별도 화면에서 따로 수행해야 한다.
- 결과적으로 관리자 페이지는 "관제판"이 아닌 "조회판" 수준에 머물러 있다.

🛠️ 해결 방안 / 제안 기능
---

관리자 페이지를 **UX flow 단위**로 8개 하위 이슈로 분리하여 순차 작업한다. 도메인을 가로지르는 cross-domain 작업이 필요하므로 도메인별이 아닌 사용자 시나리오 기반으로 묶는다.

작업은 본 Epic 이슈의 단일 브랜치/worktree에서 진행하며, 각 하위 이슈마다 `/superpowers:brainstorming` 으로 별도 설계 단계를 거친 뒤 구현한다.
