# ❗[버그][QA] 회원탈퇴, 물품삭제, 에러처리 등 전반적 버그 수정

🗒️ 설명
---

전체 QA 과정에서 발견된 6개 버그. 회원탈퇴 시 채팅방 미정리, 물품 삭제 시 FK 위반, 에러 메시지 노출, Admin 에러코드 오류, 알림 동의 연동 버그, 탈퇴 accountStatus 미변경.

🔄 재현 방법
---

**버그 1 — 회원탈퇴 시 채팅방 정리 안 됨**
1. A-B 회원 간 채팅방 생성
2. A 회원 탈퇴
3. ChatRoom 레코드 잔존 확인

**버그 2 — 물품 직접 삭제 시 FK 위반**
1. 물품에 거래 요청 (TradeRequestHistory 생성)
2. 거래 수락 → ChatRoom 생성
3. 물품 소유자가 물품 삭제
4. DB FK 제약조건 위반 발생

**버그 3 — Exception 에러 메시지 클라이언트 노출**
1. 서버 내부 오류 발생
2. Response body에 `e.getMessage()` 그대로 반환 확인

**버그 4 — Admin 신고 상세 조회 에러코드**
1. 존재하지 않는 신고 ID로 `/api/admin/reports/item-detail` 호출
2. 400 INVALID_REQUEST 반환 (404 REPORT_NOT_FOUND 이어야 함)

**버그 5 — 이용약관 동의 시 알림 동의 연동 버그**
1. 회원 가입 → 이용약관 동의 (마케팅 미동의)
2. 채팅 알림, 거래 알림도 false로 설정됨 확인

**버그 6 — 회원탈퇴 시 accountStatus 미변경**
1. 회원 탈퇴
2. DB에서 해당 회원 accountStatus 확인 → ACTIVE_ACCOUNT 그대로

✅ 예상 동작
---

- 회원 탈퇴 시 해당 회원이 속한 모든 ChatRoom 정리 (OR 조건 쿼리)
- 물품 삭제 시 TradeRequestHistory CANCELED 처리 후 Item softDelete (FK 위반 없음)
- 서버 내부 오류 시 generic 메시지만 반환 (내부 정보 미노출)
- 존재하지 않는 신고 조회 시 404 REPORT_NOT_FOUND 반환
- 이용약관 동의 시 서비스 알림(채팅/거래/활동/콘텐츠)은 항상 true, 마케팅만 선택값
- 회원 탈퇴 시 accountStatus = DELETE_ACCOUNT 로 변경
