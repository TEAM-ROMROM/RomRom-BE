📝 현재 문제점
---

- 현재 admin 대시보드(`/admin`)는 통계 카드 4개(전체 회원 / 등록 물품 / **진행중 거래** / 신고 접수)와 최근 가입 회원·최근 등록 물품 테이블 2개만 노출한다.
- 거래가 "진행중 거래" 하나로 뭉뚱그려져 있어, **교환완료(TRADED) / 취소(CANCELED) / 채팅중(CHATTING) / 완료요청(TRADE_COMPLETE_REQUESTED) / 대기(PENDING)** 같은 상태별 현황을 한눈에 볼 수 없다.
- 작성된 후기(TradeReview) 건수가 대시보드에 전혀 드러나지 않는다.
- 통계가 **전체 누적 순간값**만 있고, 기간(오늘/7일/30일/직접선택)별로 끊어 볼 수 없다.
- 카드를 눌러 해당 현황의 상세 목록으로 바로 이동(드릴다운)할 수 없어, 좌측 메뉴를 다시 찾아 들어가야 한다.
- 결과적으로 대시보드가 "관제판"이 아니라 "요약 숫자판" 수준에 머물러 있다.

🛠️ 해결 방안 / 제안 기능
---

PostgreSQL에 **이미 존재하는** 거래/회원/물품/후기 데이터를 기반으로 대시보드를 통계 관제판으로 개편한다. (실시간 접속/방문자 분석은 신규 수집 인프라가 필요하므로 별도 이슈로 분리 — 아래 "관련 이슈" 참고.)

1. **거래 상태별 통계 카드** — 기존 "진행중 거래" 단일 카드를 `TradeStatus` 5종(PENDING/CHATTING/TRADE_COMPLETE_REQUESTED/TRADED/CANCELED)으로 분리한 개별 카드로 교체. 전체 회원/등록 물품 카드는 유지.
2. **신규 후기 카운트 카드** — `TradeReview` 건수를 읽기 전용 숫자 카드로 추가(클릭 동작 없음). 후기 목록/숨김 기능은 별도 후기 관리 이슈로 분리.
3. **기간 필터** — `전체 / 오늘 / 7일 / 30일 / 직접선택(날짜 범위)` 토글. 선택 기간에 따라 카드 카운트와 하단 리스트가 갱신된다. "전체"는 누적값.
4. **카드 드릴다운** — 거래 상태 카드 클릭 시 해당 상태로 필터링된 거래 목록 페이지(`/admin/trades?tradeStatus=...&startDate=...&endDate=...`)로 이동. 회원/물품 카드도 각 목록 페이지로 이동.
5. **최근 교환완료 거래 리스트** — 하단에 최근 TRADED 거래(양쪽 물품/회원/완료시각)를 표시. 행 클릭 시 거래 상세로 이동.
6. **기존 위젯 유지** — "최근 가입 회원 / 최근 등록 물품" 테이블 2개는 그대로 두고, 위 신규 섹션을 그 위/아래에 배치.
7. **(기존 계획 유지) 시계열 통계** — 최근 N일 일별 카운트(회원/물품/거래완료/신고) 추이. 차트 라이브러리는 ApexCharts 사용(아래 분석 이슈와 동일 스택으로 통일).
