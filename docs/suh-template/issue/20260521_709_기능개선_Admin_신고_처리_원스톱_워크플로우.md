📝 현재 문제점
---

- 현재 신고 처리는 `POST /api/admin/reports/update-status`로 상태만 변경 가능하다 (PENDING → COMPLETED 등).
- 신고 상세 응답에 피신고자/신고대상 물품의 핵심 정보가 충분하지 않아, 운영자가 정지/물품삭제 등 후속 조치를 하려면 회원/물품 화면으로 이동해서 다시 작업해야 한다.
- 신고 처리 결과(어떤 admin이 어떤 신고를 어떤 액션으로 처리했는지)가 제재이력(SanctionHistory)과 명확히 연결되지 않는다.
- 결과: 한 건의 신고를 처리하는 데 화면 3~4개를 이동해야 하며, 처리 누락/이중 처리 위험이 있다.

🛠️ 해결 방안 / 제안 기능
---

신고 상세 화면에서 원클릭으로 후속 조치까지 끝낼 수 있는 워크플로우 API를 제공한다.

- 신고 상세 응답에 **피신고자 요약 + 신고대상(물품/회원) 요약 + 동일 피신고자에 대한 다른 신고 카운트** 인라인 포함
- 원클릭 액션 API: 신고를 처리하면서 동시에 정지/물품삭제/반려 중 하나 실행 → 해당 신고를 자동 COMPLETED/REJECTED 처리 + 제재이력에 신고 ID와 함께 기록
- 처리 후 알림(피신고자/신고자) 발송 정책 명확화 (별도 issue로 분리 가능하나 본 issue에서 인터페이스 정의)
