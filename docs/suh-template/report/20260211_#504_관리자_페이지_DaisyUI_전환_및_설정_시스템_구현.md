### 📌 작업 개요
관리자 페이지 전면 리디자인 (AdminLTE/Bootstrap → Tailwind CSS 4 + DaisyUI 5), RESTful API 아키텍처 전환, 신고(Report) 관리 기능 추가, DB 기반 동적 설정 관리 시스템(system_config + Redis 캐시) 도입, 코드 리뷰 이슈 수정

이슈: [#504](https://github.com/TEAM-ROMROM/RomRom-BE/issues/504) 관리자 페이지 기존 Admin4에서 DaisyUI 로 변경 필요

### 🎯 구현 목표
- AdminLTE 4 전면 제거, Tailwind CSS 4 + DaisyUI 5 기반 UI 재구성
- action 기반 단일 엔드포인트 → RESTful 개별 엔드포인트 전환

- DB 기반 AI 모델 설정 관리 시스템 (system_config 테이블 + Redis 캐시)

#### 3. RESTful API 아키텍처 전환

**제거**: action 기반 단일 엔드포인트
```
POST /admin/api/dashboard  (action=recent-members|recent-items)
POST /admin/api/items      (action=init|list|delete|detail)
```

**신규**: RESTful 개별 엔드포인트
```
GET    /api/admin/dashboard/stats           → 대시보드 통계
GET    /api/admin/dashboard/recent-members  → 최근 회원 8명
GET    /api/admin/dashboard/recent-items    → 최근 물품 8개
GET    /api/admin/items?pageNumber=&pageSize=&searchKeyword=&itemCategory=&...  → 물품 목록
DELETE /api/admin/items/{itemId}            → 물품 삭제
GET    /api/admin/members?pageNumber=&pageSize=&searchKeyword=  → 회원 목록
POST   /api/admin/reports                   → 신고 관리 (action 기반)
```

- **파일**: `AdminApiController.java`
- **이유**: 기존 action 기반 구조가 직관적이지 않고, 각 엔드포인트의 역할이 명확하지 않음

#### 4. 신고(Report) 관리 기능 추가

- **파일**: `AdminReportService.java`, `AdminReportRequest.java`, `AdminReportResponse.java`, `ReportStatus.java`
- **변경 내용**:
  - `ReportStatus` enum 추가 (`PENDING`, `PROCESSING`, `COMPLETED`, `REJECTED`)
  - `ItemReport`, `MemberReport` 엔티티에 `status` 필드 추가
  - `ItemReportRepository`, `MemberReportRepository`에 상태별 조회/카운트 메서드 추가
  - `AdminReportService`에서 목록 조회, 상세 조회, 상태 변경, 통계 기능 구현
  - `reports.html`에 통계 카드, 탭(물품/회원 신고), 상태 필터, 상세 모달 구현
- **이유**: 관리자가 사용자/물품 신고를 관리하고 상태를 추적할 수 있도록

#### 5. DB 기반 동적 설정 관리 시스템 (system_config + Redis 캐시)

- **신규 파일**:
  - `SystemConfig.java` — JPA 엔티티 (`config_key`, `config_value`, `description`)
  - `SystemConfigRepository.java` — JPA 레포지토리
  - `SystemConfigCacheService.java` — Redis Hash(`system:config`) 기반 캐시 R/W
  - `SystemConfigService.java` — DB + 캐시 조합 비즈니스 로직
  - `SystemConfigApiController.java` — Admin API 컨트롤러
  - `V1_4_8__create_system_config_table.sql` — Flyway 마이그레이션

- **설정 API**:
  ```
  GET    /api/admin/config/ai           → AI 설정 전체 조회
  PUT    /api/admin/config/ai           → AI 설정 일괄 저장 + 캐시 갱신
  POST   /api/admin/config/cache/reload → 전체 캐시 초기화 (DB → Redis)
  ```

- **데이터 흐름**:
  1. 서버 기동 → `@EventListener(ApplicationReadyEvent)` → DB 전체 조회 → Redis Hash 저장 + Properties 빈 갱신
  2. 설정 읽기 → Redis Hash에서 바로 조회 (DB 쿼리 없음)
  3. Admin에서 설정 변경 → DB UPDATE + Redis HSET + Properties 빈 런타임 갱신

- **초기 데이터** (11개 AI 설정):
  - `ai.primary.provider` = `ollama`, `ai.fallback.provider` = `vertex`
  - Ollama: base-url, chat-model, embedding-model, enabled
  - Vertex AI: generation-model, embedding-model, embedding-location, generation-location, enabled

- **이유**: application.yml에 하드코딩된 AI 설정을 서버 재시작 없이 Admin UI에서 즉시 변경 가능하도록

#### 6. settings.html — AI 설정 카드 UI

- **파일**: `settings.html`
- **변경 내용**: 기존 시스템 정보 + 테마 카드 아래에 AI 설정 카드 추가
  - Primary/Fallback AI 드롭다운 (Ollama ↔ Vertex AI 스위칭)
  - Ollama 설정: ON/OFF 토글, Base URL, 채팅 모델, 임베딩 모델
  - Vertex AI 설정: ON/OFF 토글, 생성 모델, 임베딩 모델, 생성 리전, 임베딩 리전
  - 저장 버튼 (API → DB + Redis 갱신), 캐시 초기화 버튼
  - 성공/에러 알림 표시
- **이유**: 관리자가 AI 모델 설정을 웹에서 직접 변경 가능

#### 7. 코드 리뷰 Critical 이슈 수정

- **`Member.java`**: `password` 필드에 `@JsonIgnore` 추가 → Admin API에서 비밀번호 노출 차단
- **`ItemReport.java`, `MemberReport.java`**: `@JsonIgnoreProperties({"hibernateLazyInitializer", "handler"})` 추가 → Hibernate 프록시 직렬화 오류 방지
- **`AdminReportService.java`**: `handleAction`의 `@Transactional(readOnly=true)` → `@Transactional`로 변경 → self-invocation으로 인한 `updateStatus` 쓰기 실패 방지

#### 8. Flyway 마이그레이션 — Report status 초기화

- **파일**: `V1_4_7__update_report_status_default_pending.sql`
- **변경 내용**: `item_report`, `member_report` 테이블의 `status`가 NULL인 기존 데이터를 `'PENDING'`으로 UPDATE
- **이유**: 신고 관리 기능 추가에 따라 기존 데이터의 status 정합성 보장

#### 9. members.html 전면 재작성

- **기존**: 서버사이드 Thymeleaf 렌더링 (`memberService.getAllMembers()` → 전체 회원 한번에 전달)
- **변경**: API 기반 클라이언트사이드 렌더링 (`GET /api/admin/members` → 페이지네이션/검색)
- 검색 폼 (닉네임/이메일), 페이지 크기 선택 (10/20/50), 프로필 이미지 표시

#### 10. 공통 유틸리티 보강 (common.js)

- `escapeHtml()` — XSS 방지, 모든 innerHTML 렌더링에 적용

### 🔧 주요 변경사항 상세

#### SystemConfigService.java — 설정 관리 핵심 로직
`@EventListener(ApplicationReadyEvent.class)`로 서버 기동 완료 시 자동으로 DB → Redis 캐시 로딩. `applyToProperties()` 메서드를 통해 `SuhAiderProperties`, `VertexAiProperties` 빈의 값을 런타임에 직접 갱신하여, 기존 AI 서비스 코드 수정 없이 설정 변경 즉시 반영

**특이사항**:
- Vertex AI `Client` 빈은 서버 기동 시 location 기반으로 생성. location 변경 시 서버 재시작 필요. 모델명 변경은 런타임 반영
- RomRomInitiation이 RomRom-Application 모듈에 있고 SystemConfigService가 RomRom-Web에 있어 직접 주입 불가. `ApplicationReadyEvent` 리스너로 해결

#### AdminReportService.java — @Transactional 수정
`handleAction()`이 `readOnly=true`였는데 내부에서 `updateStatus()`를 호출. Spring AOP 프록시는 self-invocation을 가로채지 못하므로 `updateStatus()`의 별도 `@Transactional`이 무시되어 readOnly 트랜잭션 안에서 쓰기 작업 실행. `readOnly` 제거하여 해결

### 📌 참고사항
- `getAllMembers()`, `getAllItems()` 메서드가 `MemberService`/`ItemService`에 아직 잔존 — 다른 곳에서 사용 여부 확인 후 정리 필요
