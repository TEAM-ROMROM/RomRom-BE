## 📌 작업 개요

차단된 회원 목록 조회 및 타인 프로필 조회 API에서 위치 주소(`locationAddress`)와 차단 상태(`isBlocked`)를 함께 반환하도록 개선

### 요청 사항
1. 차단된 회원 목록 조회 시 위치 주소 정보 포함 (예: "서울특별시 광진구 화양동")
2. 타인 프로필 조회 시 해당 회원을 차단했는지 여부(`isBlocked`) 표시

### 3. MemberBlockRepository에 단방향 차단 확인 메서드 추가

**파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/repository/MemberBlockRepository.java`

```java
@Query(value =
    "SELECT EXISTS (" +
        "    SELECT 1 FROM member_block " +
        "    WHERE blocker_member_id = :blockerId AND blocked_member_id = :blockedId" +
        ")", nativeQuery = true)
boolean existsByBlockerAndBlocked(@Param("blockerId") UUID blockerId, @Param("blockedId") UUID blockedId);
```

- 기존 `existsBlockBetween()`은 양방향 확인 (A가 B를 차단 OR B가 A를 차단)
- 신규 메서드는 단방향 확인 (A가 B를 차단했는지만 확인)
- 타인 프로필 조회 시 "내가 이 사람을 차단했는지" 확인용

### 4. MemberBlockService 수정 - 차단 목록에 위치 주소 포함

**파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/service/MemberBlockService.java`

**변경 사항**:
- `MemberLocationRepository` 주입
- 차단된 회원들의 위치 정보 일괄 조회 (N+1 방지)
- 각 Member에 `locationAddress` 세팅

```java
// 위치 정보 일괄 조회 (N+1 방지)
Set<UUID> memberIds = blockedMembers.stream()
    .map(Member::getMemberId)
    .collect(Collectors.toSet());

Map<UUID, MemberLocation> locationMap = memberLocationRepository
    .findByMemberMemberIdIn(memberIds)
    .stream()
    .collect(Collectors.toMap(
        ml -> ml.getMember().getMemberId(),
        ml -> ml
    ));

// 각 Member에 locationAddress 세팅
blockedMembers.forEach(member -> {
    MemberLocation location = locationMap.get(member.getMemberId());
    if (location != null) {
        member.setLocationAddress(location.getFullAddress());
    }
});
```

### 5. MemberService 수정 - 타인 프로필 조회 시 isBlocked/locationAddress 세팅

public MemberResponse getMemberInfoById(MemberRequest request) {
    UUID currentMemberId = request.getMember().getMemberId();
    UUID memberId = request.getMemberId();

    // ... 회원 조회 로직 ...

    if (memberLocation != null) {

## 📌 참고사항

- `locationAddress`는 `@Transient` 필드로 DB에 저장되지 않음
- `isBlocked`도 `@Transient` 필드로 API 응답 시에만 사용
- N+1 쿼리 방지를 위해 `findByMemberMemberIdIn()` 메서드로 일괄 조회
- 프로젝트 규칙에 따라 Controller → Service로 `request.setMember()` 후 `request`만 전달
