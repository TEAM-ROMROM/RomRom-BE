## 📌 작업 개요

PR Preview 빌드 시 **실시간 진행 상황을 PR 댓글로 확인**할 수 있는 기능 추가. 기존에는 빌드 완료까지 👀 리액션만 표시되어 진행 상황을 알 수 없었으나, 이제 각 단계별 진행 상태와 소요 시간을 실시간으로 확인 가능.

**주요 구현 내용**:
- 3단계 진행 상황 테이블 (프로젝트 빌드 → Docker 이미지 생성 → 서버 배포)
- 각 단계별 소요 시간 표시
- 워크플로우 실시간 로그 링크 제공
- 하나의 댓글을 UPDATE 방식으로 계속 갱신 (댓글 스팸 방지)
- 실패 시 정확한 실패 지점 표시

## ✅ 구현 내용

### 1. 진행 상황 댓글 시스템 설계

**3단계 구조 선택 이유**:
- SSH 내부에서는 GitHub API 호출이 어려움
- "서버 배포"와 "Health Check"를 하나로 통합
- 실질적으로 사용자가 인지하는 단계와 일치

| 순서 | 단계 | 위치 | 역할 |
|------|------|------|------|
| 1 | 진행 상황 댓글 생성 | PR 정보 가져오기 후 | 초기 댓글 생성 |
| 2 | 진행 상황 - 빌드 완료 | Gradle 빌드 후 | 1단계 ✅ 업데이트 |
| 3 | 진행 상황 - Docker 완료 | Docker Push 후 | 2단계 ✅ 업데이트 |
| 4 | 배포 완료 코멘트 | SSH 배포 후 | 3단계 ✅ 최종 완료 |

### 2. 추가된 Step 상세

#### Step 1: 진행 상황 댓글 생성
- **위치**: PR 정보 가져오기 직후
- **역할**: 초기 진행 상황 테이블 생성
- **출력값**: `comment_id`, `start_time`

```yaml
- name: 진행 상황 댓글 생성
  id: progress
  uses: actions/github-script@v7
```

#### Step 2: 진행 상황 - 빌드 완료
- **위치**: Gradle 빌드 직후
- **역할**: 빌드 완료 표시 + Docker 진행 중 표시
- **출력값**: `build_duration`, `docker_start`

#### Step 3: 진행 상황 - Docker 완료
- **위치**: Docker Push 직후
- **역할**: Docker 완료 표시 + 배포 진행 중 표시
- **출력값**: `build_duration`, `docker_duration`, `deploy_start`

#### Step 4: 배포 완료 코멘트 (기존 수정)
- **변경**: `createComment` → `updateComment`
- **역할**: 최종 완료 상태 표시 + Preview URL 안내

### 3. 실패 처리 개선

**실패 지점 정확히 표시**:
```javascript
if (!commentId) {
  // 진행 상황 댓글 생성 전 실패 (초기화 단계)
  buildStatus = '⚠️ 초기화 실패';
} else if (!buildDuration) {
  // 빌드 단계에서 실패
  buildStatus = '❌ 실패';
} else if (!dockerDuration) {
  // Docker 단계에서 실패
  dockerStatus = '❌ 실패';
} else {
  // 배포 단계에서 실패
  deployStatus = '❌ 실패';
}
```

### 파일: `.github/workflows/PROJECT-SPRING-SYNOLOGY-PR-PREVIEW.yaml`

#### 시간 계산 로직
```javascript
const elapsed = now - startTime;
const minutes = Math.floor(elapsed / 60000);
const seconds = Math.floor((elapsed % 60000) / 1000);
const duration = minutes > 0 ? `${minutes}분 ${seconds}초` : `${seconds}초`;
```

#### 댓글 업데이트 로직
```javascript
await github.rest.issues.updateComment({
  owner: context.repo.owner,
  repo: context.repo.repo,
  comment_id: commentId,
  body: body
});
```

#### 워크플로우 URL 생성
```javascript
const runUrl = `https://github.com/${context.repo.owner}/${context.repo.repo}/actions/runs/${context.runId}`;
```

**특이사항**:
- `commentId`는 숫자 타입으로 통일 (`${{ steps.progress.outputs.comment_id || 0 }}`)
- 실패 시 `commentId`가 없으면 새 댓글 생성으로 폴백

## 📌 참고사항

### Health Check 설정
- **하이브리드 방식**: Actuator 우선 시도 → 로그 패턴 폴백
- **타임아웃**: 최대 120초 (24회 × 5초 간격)
- **Actuator 엔드포인트**: `/actuator/health`
- **로그 패턴**: `"Started .* in [0-9.]+ seconds"`

### 리뷰 이슈 수정 사항
| 이슈 | 수정 내용 |
|------|----------|
| Critical: commentId 타입 불일치 | 숫자 타입으로 통일 |
| Major: 실패 지점 판단 오류 | 조건문 분리로 정확한 판단 |
| Minor: 초기 실패 미구분 | `⚠️ 초기화 실패` 상태 추가 |
