### 📌 작업 개요
Vertex AI 의존도를 줄이고 자체 AI 서버(ai.suhsaechan.kr)로 임베딩 생성을 이전. SUH-AIder 라이브러리를 통해 embeddinggemma:latest 모델(768차원) 사용. 기존 Vertex AI는 fallback으로 유지하여 장애 시 자동 복구 가능.

**보고서 파일**: `.report/20260104_#427_Vertex_AI_자체_AI_서버_이전.md`

---

### 🎯 구현 목표
- Vertex AI → 자체 AI 서버(https://ai.suhsaechan.kr) 이전
- 임베딩 모델: embeddinggemma:latest (768차원)
- Fallback 전략: SUH-AIder 실패 시 기존 Vertex AI 자동 사용
- 기존 Vertex AI 로직 삭제하지 않고 유지

### ✅ 구현 내용

#### 1. SUH-AIder 라이브러리 의존성 추가
- **파일**: `RomRom-Domain-AI/build.gradle`
- **변경 내용**: `api 'kr.suhsaechan:suh-aider:1.1.2'` 의존성 추가
- **이유**: 자체 AI 서버와 통신하기 위한 클라이언트 라이브러리

#### 2. SUH-AIder 설정 Properties 클래스 생성
- **파일**: `RomRom-Domain-AI/src/main/java/com/romrom/ai/properties/SuhAiderProperties.java` (신규)
- **변경 내용**: `suh.aider` prefix로 설정 값 바인딩하는 Properties 클래스 구현
- **이유**: 하드코딩 대신 application.yml에서 설정 읽어오도록 구성

```java
@ConfigurationProperties(prefix = "suh.aider")
public class SuhAiderProperties {
  private String baseUrl;
  private Security security;  // apiKey
  private Embedding embedding; // defaultModel
}
```

#### 3. EmbeddingService 임베딩 생성 로직 수정
- **파일**: `RomRom-Domain-AI/src/main/java/com/romrom/ai/service/EmbeddingService.java`
- **변경 내용**: SUH-AIder 우선 시도 → 실패 시 Vertex AI fallback 패턴 적용
- **이유**: 자체 서버 우선 사용하되, 장애 시에도 서비스 중단 없이 Vertex AI로 자동 전환

```java
private float[] generateAndExtractEmbeddingAfterNormalization(String text) {
    // 1. SUH-AIder 시도
    try {
        String embeddingModel = suhAiderProperties.getEmbedding().getDefaultModel();
        List<Double> embedding = suhAiderEngine.embed(embeddingModel, normalized);
        return CommonUtil.convertDoubleListToFloatArray(embedding);
    } catch (Exception e) {
        log.warn("SUH-AIder 임베딩 실패, Vertex AI fallback 시도: {}", e.getMessage());
    }

    // 2. Fallback: Vertex AI
    return EmbeddingUtil.extractVector(vertexAiClient.generateEmbedding(normalized));
}
```

#### 4. 임베딩 벡터 변환 유틸리티 추가
- **파일**: `RomRom-Common/src/main/java/com/romrom/common/util/CommonUtil.java`
- **변경 내용**: `convertDoubleListToFloatArray()` 메서드 추가
- **이유**: SUH-AIder 응답(List<Double>)을 기존 시스템(float[])과 호환되도록 변환

```java
public static float[] convertDoubleListToFloatArray(List<Double> doubleList) {
    if (doubleList == null || doubleList.isEmpty()) {
        return new float[0];
    }
    float[] result = new float[doubleList.size()];
    for (int i = 0; i < doubleList.size(); i++) {
        result[i] = doubleList.get(i).floatValue();
    }
    return result;
}
```

### 🔧 주요 변경사항 상세

#### application-dev.yml / application-prod.yml 설정 (gitignore 처리됨)
```yaml
# SUH-AIder 설정 (자체 AI 서버)
suh:
  aider:
    base-url: https://ai.suhsaechan.kr
    security:
      api-key: ********
    embedding:
      default-model: embeddinggemma:latest
```

#### Fallback 로직 동작 방식
1. 텍스트 정규화 (공백 처리)
2. SUH-AIder 임베딩 시도 → 성공 시 768차원 벡터 반환
3. SUH-AIder 실패 시 → Vertex AI로 자동 fallback
4. 지연시간 로깅으로 성능 모니터링 가능

**특이사항**:
- 기존 Vertex AI 로직은 삭제하지 않고 fallback으로 유지
- 헬스체크/롤백 플래그는 구현하지 않음 (사용자 요청에 따라 생략)
- 임베딩 모델명은 Properties에서 읽어오므로 환경별 설정 가능
