### 📌 작업 개요
물품 등록 시 이미지 URL의 도메인이 `file.domain` 설정값과 달라 등록이 실패하는 버그 수정.
스토리지가 NAS에서 MinIO S3로 전환된 이후 발생한 문제로, 설정값 변경이 아닌 파일 경로 추출 로직 자체를 도메인 독립적으로 수정하여 근본 해결.

**이슈**: [#570](https://github.com/TEAM-ROMROM/RomRom-BE/issues/570)

---

### 🔍 문제 분석

`FileUtil.extractFilePath(baseUrl, imageUrl)`에서 `imageUrl.startsWith(baseUrl)` 방식으로 유효성 검증.
`file.domain` 설정값이 `http://suh-project.synology.me`로 남아 있는 상태에서 MinIO S3(`https://s3.suhsaechan.kr`)로 업로드된 이미지 URL이 들어오면 startsWith 체크 실패 → `CustomException: 잘못된 파일이 요청되었습니다.` 발생.

**핵심**: `file.domain`을 어느 한 값으로 고정해도 두 도메인을 모두 지원할 수 없는 구조적 문제.

---

### ✅ 구현 내용

#### `FileUtil.extractFilePath` 로직 변경
- **파일**: `RomRom-Domain-Storage/src/main/java/com/romrom/storage/util/FileUtil.java`
- **변경 내용**: `baseUrl` startsWith 검증 제거 → URL에서 `scheme://host` 부분을 파싱하여 그 이후 경로만 추출하는 방식으로 변경
- **이유**: `file.domain` 설정값에 의존하지 않으므로 NAS/MinIO 등 어떤 도메인의 URL이든 경로 추출 가능

---

### 🔧 주요 변경사항 상세

#### 기존 방식 → 변경 방식

**기존**: `baseUrl`과 `imageUrl`의 prefix 일치 여부 검사 후 baseUrl 이후 경로 추출
```
https://s3.suhsaechan.kr/romrom/public/images/abc.webp
→ startsWith("http://suh-project.synology.me") → false → 예외 발생
```

**변경**: URL에서 `://` 이후 첫 번째 `/` 위치를 찾아 그 뒤를 경로로 반환
```
https://s3.suhsaechan.kr/romrom/public/images/abc.webp → romrom/public/images/abc.webp
http://suh-project.synology.me/romrom/images/abc.png  → romrom/images/abc.png
```

**특이사항**:
- `baseUrl` 파라미터는 기존 호출부 시그니처 호환성을 위해 유지하되 로직에서 사용하지 않음
- `application-prod.yml`의 `file.domain` 수정 불필요 — 코드 배포만으로 해결

### 🧪 테스트 및 검증
- MinIO S3 URL(`https://s3.suhsaechan.kr/...`)로 물품 등록 요청 시 정상 처리 확인
- 기존 NAS URL(`http://suh-project.synology.me/...`)도 동일하게 경로 추출 가능
