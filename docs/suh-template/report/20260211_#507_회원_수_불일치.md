### 📌 작업 개요
관리자 대시보드의 회원수와 회원 관리 페이지의 회원수 불일치 문제 해결. 쿼리 조건 차이가 원인으로, 계정 상태 필터를 추가하여 일관된 조회 제공.

**이슈**: [#507 회원 수 불일치](https://github.com/TEAM-ROMROM/RomRom-BE/issues/507)

### 🔍 문제 분석
대시보드와 회원 관리 페이지의 회원 조회 쿼리 조건이 달랐음:

| 페이지 | 쿼리 조건 |
|--------|-----------|
| 대시보드 | `isDeleted = false AND accountStatus != 'DELETE_ACCOUNT'` |
| 회원 관리 | `isDeleted = false` 만 |

`isDeleted = false`이면서 `accountStatus = 'DELETE_ACCOUNT'`인 회원이 존재할 경우, 회원 관리 페이지에서는 카운트되지만 대시보드에서는 제외되어 숫자 불일치 발생.

### ✅ 구현 내용

#### AdminRequest에 계정 상태 필터 필드 추가
- **파일**: `RomRom-Common/src/main/java/com/romrom/common/dto/AdminRequest.java`
- **변경 내용**: `AccountStatus accountStatus` 필드 추가
- **이유**: 회원 관리 페이지에서 계정 상태별 필터링 지원

#### AdminMemberDto에 계정 상태 필드 추가
- **파일**: `RomRom-Common/src/main/java/com/romrom/common/dto/AdminResponse.java`
- **변경 내용**: `AdminMemberDto`에 `String accountStatus` 필드 추가
- **이유**: 프론트엔드에서 계정 상태를 뱃지로 표시하기 위해 필요

#### Repository에 계정 상태 기반 쿼리 추가
- **파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/repository/MemberRepository.java`
- **변경 내용**: 3개 메서드 추가
  - `searchByKeyword()` - 키워드 검색 (isDeleted 필터 없음, "전체" 용)
  - `findByAccountStatus()` - 특정 계정 상태 필터
  - `searchByKeywordAndAccountStatus()` - 키워드 + 계정 상태 조합 검색
- **이유**: 필터 조합별 쿼리 메서드 필요

#### Service 필터 분기 로직 추가
- **파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/service/MemberService.java`
- **변경 내용**: `getMembersForAdmin()`에 accountStatus 유무에 따른 분기 처리
  - `accountStatus` 지정 시 → `findByAccountStatus()` / `searchByKeywordAndAccountStatus()` 사용
  - `accountStatus` 미지정 시 (전체) → `findAll()` / `searchByKeyword()` 사용
- **이유**: "전체"는 모든 회원(탈퇴 포함), 특정 상태 선택 시 해당 상태만 표시

#### 프론트엔드 계정 상태 필터 및 표시 추가
- **파일**: `RomRom-Web/src/main/resources/templates/admin/members.html`
- **변경 내용**:
  - 계정 상태 드롭다운 추가 (전체/활성/탈퇴/테스트)
  - 테이블 "상태" 컬럼을 "계정 상태"로 변경하고 `accountStatus` 기반 뱃지 렌더링
  - `renderAccountStatusBadge()` 헬퍼 함수 추출
- **이유**: 관리자가 계정 상태별로 회원을 필터링하고 상태를 시각적으로 확인 가능

### 🔧 주요 변경사항 상세

#### 쿼리 일관성 개선
기존 "전체" 조회가 `findByIsDeletedFalse()`를 사용하여 `isDeleted = false`만 체크했으나, `findAll()` / `searchByKeyword()`로 변경하여 탈퇴 회원 포함 모든 회원을 표시하도록 수정. 이를 통해:
- **전체**: 모든 회원 표시 (탈퇴 포함)
- **활성**: `ACTIVE_ACCOUNT` 회원만 → 대시보드 숫자와 일치 확인 가능
- **탈퇴**: `DELETE_ACCOUNT` 회원만
- **테스트**: `TEST_ACCOUNT` 회원만

개별 상태 합계 = 전체 카운트로 일관성 보장.

**특이사항**:
- 대시보드 회원수는 기존 `countActiveMembers()` (활성 회원만) 그대로 유지. 대시보드는 KPI 지표로 활성 회원수를 표시하는 것이 적절
- 회원 관리 페이지에서 "활성" 필터 선택 시 대시보드 숫자와 동일한 기준으로 조회되어 불일치 의문 해소 가능
