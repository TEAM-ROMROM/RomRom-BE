### 📌 작업 개요
관리자 신고 관리 페이지(`/admin/reports`) 접근 시 500 에러 발생하는 버그 수정.
`item_report` 테이블에 `status` 컬럼이 PostgreSQL DB에 존재하지 않아 Hibernate SQL 실행 실패하는 문제.
Flyway 마이그레이션 `V1_4_9`를 추가하여 누락된 컬럼을 명시적으로 생성하도록 수정.

### 🔍 문제 분석

**에러 현상**:
- `POST /api/admin/reports` 호출 시 500 Internal Server Error
- `countByStatus`, `findAllByOrderByCreatedDateDesc` 등 `status` 컬럼 참조 쿼리 전부 실패
- PostgreSQL 에러: `ERROR: column ir1_0.status does not exist`

**근본 원인**:
- `ItemReport` Entity에 `ReportStatus status` 필드가 추가되었으나 DB에 실제 컬럼이 생성되지 않음
- 기존 V1_4_7 마이그레이션이 `ALTER TABLE ADD COLUMN` 없이 `UPDATE`만 수행
- `WHEN OTHERS THEN RAISE WARNING` 예외 핸들러가 에러를 삼켜서 마이그레이션은 "성공"으로 기록됨
- `ddl-auto: update`가 컬럼을 자동 생성해야 하지만, Flyway가 먼저 실행되어 시점 문제 발생

### ✅ 구현 내용

#### Flyway 마이그레이션 V1_4_9 추가
- **파일**: `RomRom-Web/src/main/resources/db/migration/V1_4_9__add_report_status_column.sql`
- **변경 내용**: `item_report`와 `member_report` 테이블에 `status` 컬럼을 명시적으로 추가하는 마이그레이션 생성
- **이유**: `ddl-auto: update`에 의존하지 않고 Flyway 마이그레이션으로 확실하게 컬럼 생성 보장

#### CLAUDE.md Flyway 컨벤션 추가
- **파일**: `CLAUDE.md`
- **변경 내용**: Flyway 마이그레이션 작성 시 필수 규칙 명시 (테이블 존재 여부 체크 의무화)
- **이유**: 동일 문제 재발 방지를 위한 컨벤션 문서화

### 🔧 주요 변경사항 상세

#### V1_4_9__add_report_status_column.sql
`item_report`와 `member_report` 테이블에 `status` 컬럼 추가하는 마이그레이션.

처리 순서:
1. 테이블 존재 + 컬럼 미존재 확인 후 `ALTER TABLE ADD COLUMN status VARCHAR(255) NOT NULL DEFAULT 'PENDING'` 실행
2. 이미 컬럼이 존재하면 건너뜀 (멱등성 보장)
3. 기존 NULL 데이터를 `PENDING`으로 업데이트 (V1_4_7 실패 보완)
4. 모든 SQL문이 `IF EXISTS` 테이블 존재 체크로 감싸져 있음

**특이사항**:
- 모든 SQL문에 테이블 존재 여부 체크 적용 (Flyway 컨벤션 준수)
- `NOT NULL DEFAULT 'PENDING'` 설정으로 기존 행에도 기본값 자동 적용
- `EXCEPTION WHEN OTHERS` 블록으로 전체 마이그레이션 멱등성 보장

#### CLAUDE.md
Flyway 마이그레이션 컨벤션 섹션 추가:
- 테이블 존재 여부 `IF EXISTS` 체크 필수 규칙 명시
- 마이그레이션 파일 구조 및 네이밍 가이드
- 예시 패턴 포함 (컬럼 추가, 데이터 업데이트)

### 🧪 테스트 및 검증
- 배포 후 `/admin/reports` 페이지 정상 로드 확인 필요
- `item_report` 테이블에 `status` 컬럼 존재 여부 확인: `SELECT column_name FROM information_schema.columns WHERE table_name = 'item_report' AND column_name = 'status';`
- 신고 통계 API(`action: stats`) 및 신고 목록 API(`action: item-list`) 정상 응답 확인

### 📌 참고사항
- 배포 시 Flyway가 자동으로 V1_4_9 마이그레이션 실행
- 컬럼이 이미 존재하는 환경에서도 안전하게 건너뜀 (멱등성)
- `member_report` 테이블도 동일하게 처리하여 동일 문제 사전 방지
