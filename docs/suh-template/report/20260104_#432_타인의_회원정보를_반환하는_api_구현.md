# 타인의 회원정보를 반환하는 API 구현

**이슈**: [#432](https://github.com/TEAM-ROMROM/RomRom-BE/issues/432)
**작업일**: 2026.01.04
**담당자**: @Cassiiopeia

---

## 📌 작업 개요

프로필 이미지 클릭 시 해당 회원의 프로필 정보 화면으로 이동하기 위해 타인의 회원정보를 조회하는 API 추가.
기존에는 본인 정보 조회 API(`/api/members/get`)만 존재했으며, `memberId`를 파라미터로 받아 다른 회원의 정보를 조회하는 기능 구현.

---

## 🎯 구현 목표

- `memberId`를 파라미터로 받아 해당 회원 정보 조회
- 기존 `MemberResponse` 구조 그대로 재사용
- 위치 정보가 없는 회원도 에러 없이 `null`로 반환

---

## ✅ 구현 내용

### 1. MemberRequest에 memberId 필드 추가
- **파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/dto/MemberRequest.java`
- **변경 내용**: `UUID memberId` 필드 추가
- **이유**: 조회할 회원의 ID를 파라미터로 받기 위함

### 2. MemberService에 getMemberInfoById 메서드 추가
- **파일**: `RomRom-Domain-Member/src/main/java/com/romrom/member/service/MemberService.java`
- **변경 내용**: `getMemberInfoById(MemberRequest request)` 메서드 추가
- **이유**: 기존 `getMemberInfo`는 로그인한 본인 정보만 반환하므로, `memberId`로 다른 회원 정보를 조회하는 별도 메서드 필요

### 3. MemberControllerDocs에 Swagger 문서 추가
- **파일**: `RomRom-Web/src/main/java/com/romrom/web/controller/api/MemberControllerDocs.java`
- **변경 내용**: `getMemberProfile` 메서드 시그니처 및 Swagger 문서 추가
- **이유**: API 명세 문서화

### 4. MemberController에 엔드포인트 추가
- **파일**: `RomRom-Web/src/main/java/com/romrom/web/controller/api/MemberController.java`
- **변경 내용**: `POST /api/members/get/profile` 엔드포인트 추가
- **이유**: 타인 프로필 조회 API 제공

## 🔧 주요 변경사항 상세

### getMemberInfoById 메서드

```java
@Transactional(readOnly = true)
public MemberResponse getMemberInfoById(MemberRequest request) {
    UUID memberId = request.getMemberId();
    Member member = memberRepository.findById(memberId)
      .orElseThrow(() -> new CustomException(ErrorCode.MEMBER_NOT_FOUND));

    MemberLocation memberLocation = memberLocationRepository.findByMemberMemberId(memberId)
      .orElseGet(() -> {
        log.warn("회원 위치 정보 없음: memberId={}", memberId);
        return null;
      });

    List<MemberItemCategory> memberItemCategory =
        memberItemCategoryRepository.findByMemberMemberId(memberId);

    return MemberResponse.builder()
      .member(member)
      .memberLocation(memberLocation)
      .memberItemCategories(memberItemCategory)
      .build();
}
```

**특이사항**:
- 위치 정보가 없어도 에러 대신 `null` 반환 + warn 로그 남김
- 회원이 존재하지 않는 경우만 `MEMBER_NOT_FOUND` 에러 발생
- 선호 카테고리가 없으면 빈 리스트로 반환

## 📡 API 명세

### 타인 회원 프로필 조회

| 항목 | 내용 |
|-----|------|
| **엔드포인트** | `POST /api/members/get/profile` |
| **인증** | JWT 필요 |
| **Content-Type** | `multipart/form-data` |

**요청 파라미터**:
| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|-----|------|
| `memberId` | UUID | O | 조회할 회원 ID |

**응답**:
```json
{
  "member": {
    "memberId": "2d978675-0e37-4a6c-91f3-9866df0a3411",
    "nickname": "홍길동",
    "profileUrl": "https://example.com/profile.jpg",
    "totalLikeCount": 5,
    ...
  },
  "memberLocation": {
    "siDo": "경기도",
    "siGunGu": "구리시",
    "eupMyoenDong": "교문동",
    ...
  },
  "memberItemCategories": [
    {
      "itemCategory": "WOMEN_CLOTHING"
    }
  ]
}
```

**에러 코드**:
| 코드 | 설명 |
|-----|------|
| `MEMBER_NOT_FOUND` | 회원을 찾을 수 없습니다 |
